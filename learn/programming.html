Software & Firmware Fundamentals

Well-structured code is as important as well-designed hardware. Good software practices reduce bugs, make experiments reproducible, and let others build on your work.

Version Control (Git)

Version control tracks every change to your code and hardware files, lets you revert mistakes, and enables collaboration without overwriting each other's work. Git is the de facto standard.

Key Points

  • Commit small, logical changes with descriptive messages — not "fixed stuff"
  • Use branches for experimental features and merge only when tested
  • Tag releases that correspond to published data or hardware revisions
  • Never commit large binary files (recordings, images) — use Git LFS or external storage

Serial Communication Protocols

Most lab instruments communicate via serial protocols. Understanding the differences between UART, SPI, and I²C helps you choose the right interface and debug communication failures.

Key Points

  • UART — point-to-point, asynchronous; ideal for PC-to-microcontroller links
  • SPI — fast (MHz range), full-duplex; best for high-speed ADCs and DACs
  • I²C — two-wire bus, multi-device; convenient for sensors and configuration ICs
  • Always check baud rate, data bits, parity, and stop bits when debugging UART issues

Real-Time & Timing

Many neuroscience experiments require precise timing — stimulus onset, reward delivery, or synchronisation between multiple instruments. Understanding real-time constraints prevents missed events and timing jitter.

Key Points

  • Use hardware interrupts, not polling, for time-critical events
  • Avoid blocking calls (delays, serial waits) in timing-critical loops
  • Synchronise instruments with a shared TTL clock line rather than software timestamps
  • Characterise jitter with an oscilloscope — even microsecond variations can matter

Data Formats & Reproducibility

How you store experimental data determines whether it can be re-analysed months or years later. Structured, self-describing formats and clear metadata are essential.

Key Points

  • Use self-describing formats like HDF5 or NWB instead of raw binary or CSV
  • Record all experimental parameters (gain, sample rate, filter settings) alongside data
  • Use ISO 8601 timestamps and include timezone information
  • Write a README for every dataset explaining acquisition conditions and file structure
